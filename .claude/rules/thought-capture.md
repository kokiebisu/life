# クイックメモ DB（📌）

## 概要

Notion クイックメモDB は「思考のキャプチャ → 言語化 → 配置」パイプラインの入口。
スマホ等で非同期にメモし、後で `/process` コマンドで Claude と壁打ちしながら言語化・配置する。

## DB 情報

- DB ID: `30dce17f7b98803dad7afa47a1d08dd5`
- Data Source ID: `30dce17f-7b98-81c1-b798-000b54d168de`
- MCP 経由のみ使用（スクリプト非対応・環境変数なし）

| プロパティ | 型 | 用途 |
|-----------|-----|------|
| タイトル | title | メモの内容（短くてOK、ただし「何について」かはわかるように） |
| タグ | multi_select | アイデア / メモ / リマインダー |
| 日付 | date | 記録した日 |
| ステータス | select | 未処理 / 処理済み |
| リンク先 | text | 処理後の配置先ファイルパス |

## メモの書き方（ユーザー向け）

- タイトルは「何について」がわかる程度には書く
  - OK: 「福岡 家賃相場」「sumitsugi ユーザーインタビュー」
  - NG: 「気になった」「あれ」（後で思い出せない）
- 詳細があればページ本文にも書いておくと `/process` 時に役立つ
- ステータスは未設定でも OK（`/process` が自動で拾う）

## todo DB との棲み分け

- **アクションが明確**（やるべきこと） → todo DB
- **思考・気づき・アイデア**（考えたこと・覚えておきたいこと） → クイックメモ DB

## 処理フロー

```
📌 Notion でサッとメモ（スマホ等）
    ↓
/process で1件ずつ壁打ち
    ↓
言語化 → life リポジトリに配置
    ↓
ステータス「処理済み」+ リンク先を記録
```

## 注意

- クイックメモ DB はスケジュール DB ではない。Notion Calendar には表示しない
- `/process` 以外のコマンド（from:notion 等）ではクイックメモ DB を処理しない
- daily planning 時に「未処理メモが N件あるよ」と通知するのは OK
- 2週間以上放置されたメモは `/process` 時にスキップ/削除も選択肢として提示する
