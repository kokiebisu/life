#### プロジェクト1: freee福利厚生のアプリケーション開発（2024年10月〜2025年6月）

- **チーム構成:** エンジニア4名（24卒1名・25卒1名・中途1名を含む）
- **担当領域:** バックエンド／インフラ／プロダクト改善

**業務内容**

- **セルフ課金機能の開発:** 営業介在のみだった申込フローによる導入遅延・churn を解消するため、ベネフィットサービスのセルフ申込機能を開発。freee 共通決済基盤に統合し、新規事業所作成・既存事業所アップセルの2パターンで契約状態が整合するようバックエンド API と申込導線 UI を実装。リリース後、Redash でセルフ申込率の向上と churn 率の改善を確認。

- **従業員マスタ統合:** freee福利厚生が独自 DB を保有し共通マスタに乗っていなかった問題を解消。共通従業員マスタチームと連携し API の設計・立ち上げを担当。事業所作成・一括招待のバックエンドロジックを修正し、人事労務・福利厚生・ベネフィットサービス間で必須フィールドが異なる問題、既存ユーザーの後方互換も担保。リリース前後3ヶ月の導入実績平均上昇率を比較し**約20%の改善**を確認。

- **EKS クラスタのマイグレーション（リーダー）:** 福利厚生独自 EKS クラスタを人事労務クラスタへ統合。3ヶ月おきのバージョンアップ対応負荷を削減し、当番制での運用に切替。SRE と連携し integration→staging→production の段階的移行を計画・実行、ダウンタイムなしでリリース。

**使用技術:** Ruby, Ruby on Rails, TypeScript, Terraform, Kubernetes, ArgoCD, AWS, GitHub Actions

#### プロジェクト2: freee Eラーニング 新規立ち上げ プロダクトリード（2025年7月〜2026年2月）

- **チーム構成:** エンジニア3名（24卒1名・25卒1名を含む）
- **担当領域:** プロダクトリード／バックエンド／フロントエンド／QA／外部連携
- **成果:** 2026年2月12日リリース（[プレスリリース](https://prtimes.jp/main/html/rd/p/000002007.000006428.html)）

**プロダクト概要**

freee人事労務上で展開する「freee Eラーニング」。プロ講師による1,000以上のコース・6,000本の研修動画の受講管理、企業独自教材のアップロード、部署別・階層別の受講状況可視化、未受講者への自動リマインドなどを提供する SaaS。freee人事労務との従業員データ連携によりアカウント付与作業を効率化。

**業務内容**

- 実装にとどまらず、PdM と連携してプロダクトの方向性・技術戦略・チーム運営・QA 要件策定に貢献し、リリース判断まで横断的に担当
- 人事労務へのモジュール統合 vs standalone 展開を比較検討。セキュリティ要件・開発速度・責務のシンプルさを総合評価しモジュール統合を選択。Design Doc を作成し基盤チームと合意形成
- OEM 提供元との仕様策定・外部コミュニケーションを担当
- スクラムのファシリテーション（スプリント計画・チケット策定・進捗管理）でジュニア2名をリードしながら自身もバックエンド・フロントエンドを実装
- チーム全員が未経験のコードベース統合・前例のない OEM 連携という二重の不確実性、PdM とエンジニア間の対立、上場企業 OEM 先との厳格な要件調整といった組織的課題に直面。上長との 1on1on1 を自ら設計、ペアプロでドメイン知識共有、リファインメント整備を通じてチームの信頼関係を再構築しリリースを実現

**AI 活用（Claude Code）**

- **設計フェーズ:** Design Doc・PRD の壁打ちに活用。組織同期機能では従業員データの状態遷移パターンを AI と網羅的に洗い出し、見落としを設計段階で担保
- **実装フェーズ:** git worktree と組み合わせて Claude Code を複数エージェント並列運用。サブチケットに影響範囲を記載しコンフリクトリスクを最小化
- **品質・セキュリティ:** Claude GitHub Actions を CI に組み込み複数エージェントで自動チェック。CodeRabbit 併用。機密情報は社内 LLM 基盤のマスキングを前提に運用
- **チームへの展開:** `.claude` の skills/hooks を整備し PR を single responsibility で作成する command を共有。MCP サーバー（Storybook, Figma, GitHub, Slack, Confluence, Jira, 社内脆弱性診断ツール）を活用し、Design Doc・仕様書を読み込んだ状態でコンテキスト収集を自動化

**技術的チャレンジ**

- OEM プロダクトのセキュリティ要件について、freee PSIRT チームと OEM 提供元の3者間で基準調整しリリースを実現
- OEM API の Rate Limit 制約に対し、ワーカーによる非同期処理を実装。大量の従業員データ連携を進捗表示・ステータス管理付きで提供し UX を維持

**使用技術:** Ruby, Ruby on Rails, TypeScript, Terraform, Kubernetes, ArgoCD, AWS, GitHub Actions, Bugsnag, Datadog, Redash

### 2. Groundtruth（2022年7月〜2024年9月）

**事業内容:** 位置情報を活用した広告プラットフォームの開発・提供（米国）

**雇用形態:** 正社員 / バックエンドエンジニア（フルリモート）

**チーム規模:** 10名

#### プロジェクト1: 顧客ターゲットの検索機能向上・AI レコメンド機能（2024年3月〜2024年6月）

**概要:** 広告の消費者ターゲットに対するレコメンド機能をリードで開発。

**業務内容**

- バックエンドアーキテクチャの設計・技術選定から実装・テストまで一貫して担当
- 既存 Flask から FastAPI（ASGI）へ移行し並列処理性能を改善
- 未経験ユーザー向け Top 5 Popular（cron で月次集計）と経験ユーザー向け Personalized Suggestions（AI 基盤チームの LLM 連携）の2段階設計を採用
- ML エンドポイント障害時に Top 5 Popular へ自動切替するフォールバック設計
- Elasticsearch のインデックス作成・クエリ最適化、Redis キャッシュと ElastiCache クラスタ構築（DevOps と協力）、ECS / S3 / RDS を組み合わせたスケーラブルなアーキテクチャ、Terraform による IaC 化
- API テスト・ユニットテスト・インテグレーションテストを整備

**成果**

- キャンペーン作成時の離脱を **20% 削減**（Google Analytics 計測）
- 広告インプレッションを **10% 以上向上**
- レスポンスタイム **1秒以内**を達成
- 実装は締切より **2週間早く完了**。PM のスコープ拡大提案も技術分析の上で優先順位を再調整しプロジェクトを軌道に戻した

**使用技術:** Python, Go, TypeScript, React, SCSS, Elasticsearch, Redis, AWS (ECS, S3, RDS, ElastiCache, OpenSearch), Docker, PostgreSQL, MySQL

**技術とリーダーシップの両立**

フルスタックエンジニアとして約5年、実装・設計・チームリードを横断して経験してきました。Coinmiles では参画6ヶ月で CTO とリードエンジニアが同時退職するという状況に直面し、残り3名のチームでバックエンド・インフラのリーダーシップを引き受けました。技術判断・タスク管理・採用支援・社外インテグレーターとの連携を担いながら、CI/CD ビルド時間を 70% 短縮するなど開発効率の改善にも取り組みました。

freee では Eラーニングチームのプロダクトリードとして、OEM 提供元・freee PSIRT チームを含む複数ステークホルダー間のセキュリティ要件調整を主導。チーム全員が未経験のコードベースへの統合と前例のない OEM 連携という二重の技術的不確実性、PdM・エンジニア間の対立、上場企業である OEM 先との厳格な要件調整といった組織的課題に直面しながらも、24卒・25卒を含むエンジニア3名のチームをまとめ、「freee Eラーニング」を 2026 年 2 月にリリースしました。
