## 事前に仕込む知識（Goと技術的深さ）

### Go エラーハンドリング（彼らの実装を理解して語れるようにする）

Resilireは「MyError」カスタム型を作り、`errors.As()` で型安全にエラーを判定している。
- `errors.As()` vs `errors.Is()` の違いを話せるようにする
- typed nil 問題（`*MyError` を返すと nil でも interface が nil にならない）を知っておく
- レイヤごとのエラー変換（BFF → HTTP ステータスマッピング）

### Go goroutine 安全性
- `sync.Mutex` / `sync.RWMutex` の使い所
- channel でのデータ受け渡し
- 彼らが「スレッドセーフ問題でライブラリのdocを確認する文化」を持つ → 「ドキュメント確認の重要性」を語れる

### DB / ORM
- Resilireは **sqlc** を使用（SQLを先に書いてGoコードを自動生成）
- 面接で「GoのDB操作は？」と聞かれたら sqlc を知っているとプラスになる
- 「SQLを先に書くので意図しないN+1が起きにくい」という設計思想と繋げて語れる

### PostgreSQL（彼らが踏んでいる落とし穴を知っておく）
- cursor-based pagination vs offset（PK ソートが安定する理由）
- CSV COPY の NULL 文字列リテラル問題
- RLS（Row Level Security）の概念
- ENUM 型の追加は安全だが同トランザクション内での使用に注意

### GCP の補完方法
「AWS は深く使っていますが、GKE と EKS はほぼ同じ K8s の抽象化で、Terraform も同じ構成で書けます。Cloud SQL は RDS と同等の操作感です。」
→ 正直に「AWS 経験がメイン」と言いつつ「概念は同じなので移行コストは低い」に着地

Resilire は東京・大阪2拠点で「有事にも止まらない」を至上命題にしている。

> 「BCP観点でクラスタを統合し、ダウンタイムゼロで移行しました。Resilireさんの東京・大阪マルチクラスタ構成への考え方と重なるものを感じています」

### Story 1（Recommendation Microservice） ← **設計思想の深さで刺さる**
