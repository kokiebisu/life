# Algorithms - アルゴリズム学習

CS基礎・面接対策・実務応用の3つの目的を統合したアルゴリズム学習ロードマップ。

## 学習方針

各トピックで以下の3つの視点を意識する：

- **CS基礎**: 計算量分析、理論的背景、正しい実装
- **面接対策**: 典型パターンの習得、LeetCode等での演習
- **実務応用**: プロダクション環境での使いどころ

## Phase 1: 基礎データ構造

理論と実装の両方を固める。全アルゴリズムの土台。

- [ ] **配列と文字列** - 走査、in-place操作、部分配列
  - 実務: バッファ管理、テキスト処理
  - 演習: Two Pointers, Sliding Window パターン
- [ ] **連結リスト** - 単方向・双方向、番兵ノード
  - 実務: LRUキャッシュ、undo履歴
  - 演習: 反転、環検出、マージ
- [ ] **スタックとキュー** - LIFO/FIFO、単調スタック
  - 実務: コールスタック、タスクキュー、括弧マッチング
  - 演習: 有効な括弧、次に大きい要素
- [ ] **ハッシュテーブル** - 衝突解決、負荷率、ハッシュ関数設計
  - 実務: キャッシュ、インデックス、重複排除
  - 演習: Two Sum系、アナグラム、頻度カウント

## Phase 2: 木構造とヒープ

階層データの表現と効率的な検索・優先度管理。

- [ ] **二分木と二分探索木** - 走査（前順・中順・後順・レベル順）、バランス
  - 実務: DOM、ファイルシステム、データベースインデックス
  - 演習: 深さ、パス合計、LCA、検証
- [ ] **ヒープ / 優先度キュー** - max-heap, min-heap、heapify
  - 実務: タスクスケジューリング、イベント駆動処理、Top-K
  - 演習: K番目に大きい要素、メディアン、マージK個のリスト
- [ ] **Trie（接頭辞木）** - 挿入、検索、接頭辞マッチ
  - 実務: オートコンプリート、スペルチェック、IPルーティング
  - 演習: 単語検索、接頭辞一致

## Phase 3: グラフ

ネットワーク、依存関係、経路探索の基盤。

- [ ] **グラフ表現と走査** - 隣接リスト/行列、BFS、DFS
  - 実務: SNSの友達関係、ページリンク、依存解決
  - 演習: 島の数、クローン、接続成分
- [ ] **最短経路** - Dijkstra、Bellman-Ford
  - 実務: ナビゲーション、ネットワークルーティング
  - 演習: 重み付きグラフの最短経路
- [ ] **トポロジカルソート** - Kahn's algorithm、DFS方式
  - 実務: ビルドシステム、タスク依存関係、パッケージマネージャ
  - 演習: コーススケジュール
- [ ] **Union-Find** - 経路圧縮、ランクによる結合
  - 実務: ネットワーク接続性、画像セグメンテーション
  - 演習: 連結成分、冗長辺の検出

## Phase 4: コアアルゴリズムパターン

面接で最も頻出するアルゴリズム手法。

- [ ] **ソート** - マージソート、クイックソート、計数ソート、安定性
  - 実務: データベースのORDER BY、表示順序
  - 演習: 各ソートの実装、比較分析
- [ ] **二分探索** - 基本形、境界探索、解の二分探索
  - 実務: バージョン管理bisect、検索エンジン、ページネーション
  - 演習: 回転ソート配列、最小値探索
- [ ] **再帰とバックトラッキング** - 状態空間探索、枝刈り
  - 実務: 設定の組み合わせ生成、パーミッション
  - 演習: 順列、組み合わせ、N-Queens、数独
- [ ] **動的計画法** - メモ化、ボトムアップ、状態遷移の設計
  - 実務: 最適化問題、テキスト差分（diff）、リソース配分
  - 演習: ナップサック、LCS、コイン問題、編集距離
- [ ] **貪欲法** - 貪欲選択性質、最適部分構造
  - 実務: スケジューリング、圧縮（ハフマン）、近似アルゴリズム
  - 演習: 区間スケジューリング、ジャンプゲーム

## Phase 5: 実務寄りの応用知識

プロダクション環境で役立つアルゴリズム知識。

- [ ] **文字列マッチング** - KMP、Rabin-Karp
  - 実務: テキスト検索、ログ解析、正規表現エンジン
- [ ] **Consistent Hashing** - リング、仮想ノード
  - 実務: 分散キャッシュ、ロードバランシング
- [ ] **Rate Limiting アルゴリズム** - Token Bucket、Sliding Window Counter
  - 実務: API rate limiting、DDoS防御
- [ ] **Bloom Filter** - 確率的データ構造、偽陽性
  - 実務: キャッシュフィルタ、スパム検出、重複チェック
- [ ] **計算量の実践的理解** - Big-O だけでなく定数係数、キャッシュ局所性
  - 実務: パフォーマンスチューニング、ボトルネック分析

## 進め方

1. **Phase順に進める必要はない** - 興味や面接準備の優先度に応じて柔軟に
2. **各トピック**: 理論理解 → 自力実装 → 演習問題 → 実務での活用を考える
3. **実装言語**: 得意な言語で（面接対策なら面接で使う言語で）
4. **記録**: 学んだことは `notes/` ディレクトリに、振り返りは日記に
5. **迷ったら**: `/ask:study` で技術メンター（Fowler, Hightower, Larson）に相談
