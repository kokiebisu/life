# People

人物プロファイルシステム。教会・家族・友人・仕事仲間など全ての人を管理する。

ファイル構造:
- `aspects/people/me.md` — 自分自身（特別ファイル）
- `aspects/people/family/<英語名>.md` — 家族・兄弟姉妹
- `aspects/people/church/<教会名>/<地域>/<英語名>.md` — 教会メンバー
- その他は `aspects/people/<カテゴリ>/<英語名>.md`

## relation ごとのテンプレート

### relation: church または family（祈り記録あり）

```markdown
# [Name]

relation: church
[関係性・背景 1〜2行]

## プロフィール

- **年齢**: （例: 20代前半）
- **国籍・出身**: （例: 日本 / カナダ育ち）
- **職業・学業**: （例: 会社員 / 大学院生）
- **信仰**: クリスチャン。（特徴を1文で）
- **性格・特徴**: （例: 穏やか、リーダーシップがある）
- **趣味・好き**: （例: 音楽、アウトドア）
- **家族構成**: （例: 既婚・子供なし / 一人っ子）
- **現在のシーズン**: （今の状況・課題）
- **Kenとの関係**: （いつ・どこで知り合ったか、どんな場面で話すか）

---

## 祈り記録

### [タイトル]（開始: YYYY-MM-DD）
**ステータス:** Active / Answered

[祈りの内容]

**更新:**
- YYYY-MM-DD: [変化・近況]

---

## 出来事・記録

- YYYY-MM-DD: [出来事]
```


## 人物間リンクルール

- ファイル内で他の人物名が登場する場合、その人物のファイルへの **相対パスリンク** を付ける
- 形式: `[名前](相対パス)` — 例: `[Shiori](shiori.md)` / `[みどりちゃん](midori.md)`
- ちゃん/さん/くん などの敬称もリンクに含める: `[みどりちゃん](midori.md)`
- 自分自身のファイル名はリンクしない
- 新しく人名が登場したら、既存のリンクがなければ追加する

## 参照ルール（厳守）

**人物について質問・相談を受けたら、該当ファイルを全文読んでから答える。** 部分的に読んで回答しない。

---

## 編集ルール

**出来事・記録:**
- その人に関する新しい情報を知ったら日付付きで追記する
- **会話中に新情報が出たらその場で即座に追記する（厳守）。** 会話が終わってからまとめて追記しない
- **時系列の確認（厳守）:** 出来事を記録する際、「今起きた」のか「以前起きたことを今知った」のかを区別する。不明な場合はユーザーに確認する（例: ゴルフで赤ちゃんを見た → 「生まれた」ではなく「確認した」）
- **関連人物の波及チェック（厳守）:** 人物ファイルを修正・追記したら、リンクされている関連人物のファイルにも同じ情報がないか確認し、一緒に更新する

**祈り記録（relation: church または family を含む場合）:**
- 祈りが答えられたら: ステータスを `Answered` に変更し、`**更新:**` に「答えられた: [内容]」を追記
- 新しい祈りが始まったら: 新しい `### [タイトル]（開始: YYYY-MM-DD）` セクションを追加
- church の場合は `aspects/church/prayer-requests.md` の Active/Answered テーブルも同時に更新する

**`aspects/people/` ファイルを編集した後は `/to-notion` を実行しない**（Notion 同期対象外）
