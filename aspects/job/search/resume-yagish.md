**プロジェクトまたは業務名**

freee福利厚生のアプリケーション開発

**期間**

2024 年 10 月 〜 2025 年 6 月

**プロジェクトや業務内容の説明**

人事労務領域の福利厚生プロダクトチーム（エンジニア4名／24卒・25卒・中途を含む）でバックエンド・インフラを中心に担当。

【セルフ課金機能の開発】
営業介在のみの申込フローによる導入遅延・churn を解消するため、ベネフィットサービス（OEM 提供元の福利厚生プログラム）のセルフ申込機能を開発。freee 共通決済基盤に統合し、新規事業所作成・既存事業所アップセルの 2 パターンで契約状態が整合するようバックエンド API と申込導線 UI を実装。リリース後は Redash でセルフ申込率・churn 率を週次でモニタリングする運用を構築しました。

【従業員マスタ統合】
freee福利厚生が独自 DB を保有し共通マスタに乗っていなかった問題を解消。共通従業員マスタチームと連携し API の設計・立ち上げを担当。事業所作成・一括招待のバックエンドロジックを修正し、人事労務・福利厚生・ベネフィットサービス間で必須フィールドが異なる問題、既存ユーザーの後方互換も担保。リリース前後3ヶ月の導入実績平均上昇率を比較し約20%の改善を確認。

【EKS クラスタのマイグレーション（リーダー）】
福利厚生独自 EKS クラスタを人事労務クラスタへ統合。3ヶ月おきのバージョンアップ対応負荷を削減し、周辺プロダクトチーム群との当番制運用に切替。SRE と連携し integration→staging→production の段階的移行を計画・実行、ダウンタイムなしでリリース。

【使用技術】
Ruby, Ruby on Rails, TypeScript, Terraform, Kubernetes, ArgoCD, AWS, GitHub Actions

- 実装は締切より2週間早く完了。PM のスコープ拡大提案も技術分析の上で優先順位を再調整しプロジェクトを軌道に戻した

## → 自己 PR

**技術とリーダーシップの両立**

実装・設計・チームリードを職場ごとに担当してきました。Coinmiles では参画 6 ヶ月で CTO とリードエンジニアが同時退職した状況で、残り 3 名のチームでバックエンド・インフラのリード役を引き継ぎ、CI/CD ビルド時間を 70% 短縮しました。Groundtruth では広告レコメンド機能のバックエンド設計をリードし、Elasticsearch・Redis・AWS マネージドサービスでスケーラブルなアーキテクチャを構築。レスポンスタイム 1 秒以内を達成し、実装は締切より 2 週間早く完了しました。freee では新規プロダクト「freee Eラーニング」のリードとして、コードベース統合と OEM 連携をどちらも初回で進めながら、上場企業 OEM 先との要件合意を形成しリリースまで持ち上げました。

**AI を取り込んだ開発スタイル**

Claude Code を本格運用したのは freee Eラーニングが初めてで、3 名チームの開発フローに導入しながら手探りで標準化を進めました。設計の壁打ち・実装の並列化・自動レビューはチーム全員が同じ手段で扱えるよう、.claude の skills/hooks に集約してドキュメント化。次の組織でも同じ整備をゼロから設計できるようになりました。

**エンジニアリングへの姿勢と次のキャリア**

クラウドネイティブなアーキテクチャ設計と、Terraform による IaC・CI/CD の整備が得意領域です。カナダ育ちで英語はネイティブレベル、英語での仕様策定・外部コミュニケーション・海外チームとの協業も対応してきました。
