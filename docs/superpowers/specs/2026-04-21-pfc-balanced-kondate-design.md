# PFCバランス献立設計

> 週次PFCトラッキングを `/kondate` と `/meal` に組み込み、栄養バランスの取れた献立を自動提案する。

## 背景

現状の献立システムはカロリー（kcal）のみ追跡しており、たんぱく質・脂質・炭水化物のバランスは管理されていない。直近のログでは1日1〜2食で牛タン+玄米パック中心、野菜がほぼ入らない状態が続いている。

**目的:** 週単位でPFC目標を設定し、`/kondate` が残り日数から逆算して栄養バランスの取れたメニューを優先的に提案する仕組みを作る。

## PFC目標値

身体データ: 男性 30歳、167cm、60kg、活動量: 軽め（デスクワーク + ジム週2-3回）

**算出根拠:**
- 基礎代謝（Mifflin-St Jeor式）: 1,489 kcal
- TDEE（活動係数1.375）: 2,047 kcal
- 減量目標（-300kcal）: 約 1,750 kcal/日

**PFC配分（減量・高たんぱく）:**

| 栄養素 | 割合 | 日あたり | 週あたり |
|--------|------|---------|---------|
| たんぱく質 (P) | 30% | 131g (525kcal) | 920g |
| 脂質 (F) | 25% | 49g (438kcal) | 340g |
| 炭水化物 (C) | 45% | 197g (788kcal) | 1,380g |
| **合計** | | **1,750 kcal** | **12,250 kcal** |

目標値は体重変化に応じて `nutrition-targets.md` で手動更新する。

## 食事頻度

1日3食（朝・昼・夕）を目指す。デフォルト時間帯:
- 朝食: 08:00〜09:00
- 昼食: 12:00〜13:00
- 夕食: 19:00〜20:00

## 変更 3: `/kondate` スキル拡張

### Step 0（新規）: 今週のPFC実績を集計

`/kondate` 実行時、献立提案の前に以下を実行:

1. `aspects/diet/nutrition-targets.md` からPFC目標を読み込む
2. 今週（月〜日）の `aspects/diet/daily/*.md` を全て読む
3. 各食事のPFC値を合計して「今週の実績」を算出
4. 週目標との差分から「残り日数で1日あたり必要なPFC」を逆算

**表示フォーマット:**

```
【今週の栄養バランス】（月〜火 実績 / 水〜日 残り）
        実績    週目標   残り5日で1日あたり
P:     50g    920g    → 174g/日（通常より多め）
F:     30g    340g    → 62g/日（通常どおり）
C:    100g   1,380g   → 256g/日（通常どおり）
kcal:  900   12,250   → 2,270 kcal/日
```

5. 不足が大きい栄養素を特定し、それを補う食材・メニューを優先的に提案

### 提案フォーマット拡張

既存の提案フォーマットにPFC情報を追加:

```
【月 夕食】鶏むね肉の塩麹焼き + 玄米（クラシル）
  材料: 鶏むね肉 200g、塩麹 大さじ1、玄米パック 1食
  推定: ~520 kcal | P: 45g | F: 8g | C: 55g
  在庫: ⚠ 不足: 鶏むね肉、塩麹
  PFC: この食事でPが目標の34%カバー
```

### メニュー優先ロジック

不足栄養素に応じてメニューの優先度を調整:
- たんぱく質不足 → 鶏むね肉・卵・豆腐・魚を優先
- 脂質過多 → 蒸し・茹で系を優先、揚げ物・炒め物を避ける
- 炭水化物不足 → 玄米・オートミールをしっかり組む

## 変更 4: `/meal` スキル拡張

食事記録時にPFCを概算して daily ファイルに書き込む。

**概算基準:**

| 食事タイプ | 概算方法 |
|-----------|---------|
| 自炊（レシピあり） | レシピの材料からPFCを算出 |
| 残り物・簡易 | 主要食材から概算 |
| コンビニ | 一般的な栄養表示ベース（おにぎり P:4g F:2g C:38g 等） |
| 外食 | 記録しない（`P: — | F: — | C: —`）。週次集計では除外 |

外食が週3回以上ある場合は警告を出す:「今週は外食3回のためPFC追跡の精度が低いです」
