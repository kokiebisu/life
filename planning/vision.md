# Life Management System - ビジョン

> 2026-02-09 壁打ちセッションで策定

## コンセプト

日記を読んで、チームが理解して、明日のタスクが進化する。
静的なチェックリストではなく、自分と一緒に成長する生きたシステム。

## 優先度

```
tsumugi開発 > 運動/減量 > ギター > 投資 > study > 読書 > 福岡検討
```

## システム構成

### データフロー

```
┌─ INPUT ─────────────────────────────┐
│ Google Calendar（既存の予定）         │
│ 日記・つぶやき（mood, 進捗, 気づき） │
│ Beads（各aspectのタスク+進捗）       │
└──────────────┬──────────────────────┘
               ↓ AIが統合・判断
┌─ PROCESS ───────────────────────────┐
│ 松本あかりが全aspectを横断的に整理   │
│ - 既存予定の空き時間にタスクを配置   │
│ - 日記から mood/進捗を読み取り調整   │
│ - 各チームが具体的タスクを提案       │
│ - 優先度に基づいてバランス配分       │
└──────────────┬──────────────────────┘
               ↓
┌─ OUTPUT ────────────────────────────┐
│ Google Calendar に書き込み           │
│ → Apple Calendar に自動反映          │
│ Beads チケット更新                   │
│ Linear 同期                          │
└─────────────────────────────────────┘
```

### 技術スタック

| コンポーネント | 役割 |
|---------------|------|
| Beads (.beads/) | デイリータスク管理（Gitネイティブ） |
| Google Calendar API | カレンダー読み書き |
| Apple Calendar | Google経由で自動同期 |
| Linear | 中長期マイルストーン |
| Bun scripts | プラン生成、同期スクリプト |
| GitHub Actions | 毎朝自動実行 |

### 特殊機能

- **guitar**: レッスンプランの自動生成（3人のチームが進捗に応じて次のレッスンを設計）
- **diet**: 献立提案（栄養士の陳先生 + シェフの佐藤さん） + 運動メニュー組み立て（トレーナーの高橋コーチ）

## 3ヶ月ロードマップ

- Month 1（セットアップ期）: 生活リズム確立、ジム入会、各aspect始動
- Month 2（加速期）: 習慣定着、tsumugi MVP、投資ポジション構築
- Month 3（成果期）: -5kg達成、tsumugiリリース、起業準備具体化
