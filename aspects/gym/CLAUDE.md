# Gym Aspect

このディレクトリはジムセッションの記録と管理を担当します。

## ディレクトリ構成

| パス | 内容 |
|------|------|
| `logs/YYYY-MM-DD.md` | ジムセッション実績ログ |
| `gyms/<chain>/<location>.md` | ジムのマシン一覧・設備情報 |
| `profile.md` | ジム会員情報・個人目標 |
| `../goal.md` | 共有目標（diet と gym が参照） |

## メニュー生成

静的なメニューファイルは持たない。`/gym plan` 実行時に以下を動的参照してメニューを決定する:

1. `logs/` の直近3日のログ → 前回の種目・重量・フィードバック
2. `gyms/<location>.md` → 利用可能なマシン一覧
3. `../goal.md` → 現在の目標（重量・体組成）
4. `../daily/diary/YYYY-MM-DD.md` → 当日のコンディション（`## コンディション` セクション）
   - `low` → 全体的に軽め。重量・セット数を落とす
   - `normal` → 通常プラン
   - `high` → 重量・セット数を増やしてしっかり攻める
   - セクションなし → `normal` 扱い

## 重量調整ルール（厳守）

`/gym plan` 実行時、前回ログのフィードバックに基づいて**必ず**重量を見直す:

- `余裕` → 次回は +5kg（または回数を増やす）
- `まあまあ` → 現状維持
- `無理` / `きつい` → 次回は -5kg
- **フィードバック未記入の種目はデフォルトで `余裕` 扱いとする**（毎回記入不要）

全種目を横断して確認すること。一種目でも見落としたら失敗。

---

## 補助種目のローテーション（厳守）

メニュー決定時、**直近3セッション分の Notion ジム DB を確認し**、補助種目が重複しないよう選ぶ。

- **対象**: メイン種目（ダンベルプレス・フィックスドプルダウン・スクワットマシン・ウォーキング）以外の全種目
- **ルール**: 直近3セッション以内に使った補助種目は原則使わない
- **選択肢**: Seated Leg Curl / Prone Leg Curl / Leg Extension / Inner・Outer Thigh / Shoulder Press / Low Row / Lat Pull Down / Abdominal など、その日の部位バランスに応じて選ぶ

例: 前回・前々回でアブドミナルを使っていたら → Seated Leg Curl や Shoulder Press を選ぶ

## 種目プリファレンス（厳守）

- **胸**: ダンベルプレスを使う（ベンチプレスはケガリスクがあるため使わない）
- **脚**: スクワットマシンを使う（フリーウエイトスクワットは使わない）
- **デッドリフト**: 使わない
- **基本方針**: BIG3（ベンチプレス・スクワット・デッドリフト）にこだわらない。マシン優先で組む

`logs/YYYY-MM-DD.md`:
