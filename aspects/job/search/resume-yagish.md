#### プロジェクト1

**プロジェクトまたは業務名**

freee福利厚生のアプリケーション開発

**期間**

2024年10月〜2025年6月

**プロジェクトや業務内容の説明**

人事労務領域の福利厚生プロダクトチームで、バックエンド・インフラを中心に担当しました。主に、ベネフィットサービスのセルフ申込機能、freee共通従業員マスタとの統合、福利厚生独自EKSクラスタの人事労務クラスタ移行を担当しました。

【使用技術】
Ruby, Ruby on Rails, TypeScript, Terraform, Kubernetes, ArgoCD, AWS, GitHub Actions

#### プロジェクト2

**プロジェクトまたは業務名**

freee Eラーニング 新規プロダクト立ち上げ（開発リード）

**期間**

2025年7月〜2026年2月

**プロジェクトや業務内容の説明**

freee人事労務上で展開する新規SaaS「freee Eラーニング」の開発リードを担当しました（エンジニア3名／24卒・25卒を含む）。2026年2月12日のリリースまで、要件調整・設計・実装・QA・リリース準備を一貫して推進しました（プレスリリース: https://prtimes.jp/main/html/rd/p/000002007.000006428.html）。

当初は福利厚生の一環として検討されていましたが、人事労務データを使って対象者に必須研修を割り当てるニーズへ構想がシフトしたため、福利厚生配下・人事労務配下・スタンドアローンの3案を比較し、人事労務の周辺プロダクトとして統合する方針を選定しました。少人数チームかつOEM提供元もリアーキ中だったため、既存基盤・コードガイドライン・当番制運用に乗ることで初期開発と保守コストを抑える判断をしました。

Design Docを作成し、基盤チーム・社内セキュリティチーム・OEM提供元との要件調整を主導。OIDC連携、従業員同期、OEM APIのRate Limitを考慮した非同期処理、進捗表示、リリース前QAを担当しました。スプリント計画・チケット分解・ペアプロを通じて、ジュニア2名を含むチームの開発もリードしました。

【使用技術】
Ruby, Ruby on Rails, TypeScript, Terraform, Kubernetes, ArgoCD, AWS, GitHub Actions, Bugsnag, Datadog, Redash

#### プロジェクト1

**プロジェクトまたは業務名**

顧客ターゲットの検索機能向上・AIレコメンド機能の開発（リード）

**期間**

2024年3月〜2024年6月

**プロジェクトや業務内容の説明**

広告ターゲット選定の検索・レコメンド機能を提供する新規マイクロサービスをリード設計・実装しました。既存のAds Manager（Flaskモノリス）から疎結合に切り出し、Goで新規構築しました。

入力時はElasticsearch検索を必ず実行し、経験ユーザーには機械学習によるパーソナライズ候補も並列に取得して結果をマージする設計にしました。機械学習エンドポイントの前段にRedisキャッシュを置き、入力ごとの不要なML呼び出しを抑制。キャッシュヒット時は約300ms、初回キャッシュミス時もP99 1秒以内で返却できるようにしました。

キャンペーン作成時の離脱を20%削減し、広告インプレッションを10%増加。実装は締切より2週間早く完了し、PMのスコープ拡大提案に対して技術面から優先順位を整理しました。

【使用技術】
Go, Python, TypeScript, React, SCSS, Elasticsearch, Redis, AWS (ECS, S3, RDS, ElastiCache, OpenSearch), Docker, PostgreSQL, MySQL

## → 自己PR

**技術とリーダーシップの両立**

実装・設計・チームリードを職場ごとに担当してきました。Coinmilesでは参画6ヶ月でCTOとリードエンジニアが相次いで退職した状況で、残り3名のチームでバックエンド・インフラのリード役を引き継ぎ、CI/CDビルド時間を70%短縮。Groundtruthでは広告レコメンド機能のバックエンド設計をリードし、Elasticsearch・Redis・AWSマネージドサービスでスケーラブルなアーキテクチャを構築。レスポンスタイム1秒以内を達成し、実装は締切より2週間早く完了しました。freeeでは新規プロダクト「freee Eラーニング」の開発リードとして、コードベース統合とOEM連携を進めながら、OEM提供元・社内関係者との要件合意を形成しリリースに至りました。

**開発プロセスを整える力**

freee Eラーニングでは、3名チームで新規プロダクトを進めるため、設計レビュー・チケット分解・PRレビュー・リリース前QAの流れを整備しました。Claude CodeなどのAIツールも活用しましたが、目的はツール導入そのものではなく、仕様の見落としを減らし、小さなPRで継続的に開発できる状態を作ることでした。次の組織でも、チームの規模や開発文化に合わせて、再現性のある開発フローを設計できます。

**エンジニアリングへの姿勢と次のキャリア**

クラウドネイティブなアーキテクチャ設計と、TerraformによるIaC・CI/CDの整備が得意領域です。カナダでの学業・就業経験があり、英語での仕様策定・外部コミュニケーション・海外チームとの協業も対応してきました。

freeeで新規プロダクトをリリースまで持ち上げる経験を経て、次は腰を据えて長期で関われる自社プロダクトを軸に働きたいと考えています。リモート主体の環境で、技術判断とプロダクトの方向性の両方に関与できるポジションを探しています。
