## 職務経歴

### freee株式会社（2024/10〜2026/2）

freee人事労務のプロダクト開発に従事。前半は福利厚生チームのエンジニア、後半はEラーニングチームのプロダクトリードを担当。

#### プロジェクト1: freee福利厚生のアプリケーション開発（2024/10〜2025/6）

**チーム規模:** エンジニア4名（24卒1名・25卒1名・中途1名を含む）

**業務詳細**

**セルフ課金機能の開発**

営業介在のみの申し込みフローによる導入遅延・churnを解消するため、ベネフィットサービスのセルフ申し込み機能を開発。freee共通決済基盤に統合し、新規事業所作成・アップセル（既存事業所への追加）の2パターンで契約状態が整合するようバックエンドAPIと申し込み導線UIを実装。リリース後はRedashで**セルフ申し込み率の向上・churn率の改善**を確認。

**従業員マスタ統合**

freee福利厚生が独自DBを保有しfreee共通マスタに乗っていなかった問題を解消。人事労務などと共通の従業員マスタに統合することで、人事労務からのアップセル時にワンクリックで従業員追加が可能になった。共通従業員マスタチームと連携してAPIの設計・立ち上げを担当し、事業所作成・一括招待機能のバックエンドロジックを修正。人事労務・福利厚生・ベネフィットサービス間で必須フィールドが異なる問題や、既存ユーザーの後方互換（移行前後パターン別のUX設計）を担保しながらリリース。**リリース前後3ヶ月の導入実績平均上昇率をRedashで比較し、約20%の改善を確認。**

**EKSクラスタのマイグレーション（リーダー）**

福利厚生が独自EKSクラスタを保有しており、3ヶ月おきの独立したバージョンアップグレード対応がリソース・学習コストの負担になっていた。人事労務クラスタへの統合により、周辺プロダクトチーム群との当番制での運用に切り替え、開発への集中とクラスタ権限のシンプル化を実現。SREチームと連携し、integration→staging→productionの段階的移行計画を策定。Terraform・Kubernetesマニフェスト・YAMLの修正を担当し、段階的デプロイによりダウンタイムなしでリリース。

#### プロジェクト2: freee Eラーニングの新規立ち上げ プロダクトリード（2025/7〜2026/2）

**チーム規模:** エンジニア3名（24卒1名・25卒1名を含む）

**概要**

freee人事労務上で「freee Eラーニング」の立ち上げのプロダクトリードを担当。**2026/2/12にリリース。**（[プレスリリース](https://prtimes.jp/main/html/rd/p/000002007.000006428.html)）

プロ講師による1,000以上のコース・6,000本の研修動画の受講管理、企業独自教材のアップロード、部署別・階層別の受講状況可視化、未受講者への自動リマインドなどの機能を持つSaaS。freee人事労務との従業員データ連携により、アカウント付与作業を効率化。

**業務詳細**

エンジニアとしての実装にとどまらず、プロダクトマネージャーと連携しながらプロダクトの方向性・技術戦略・チーム運営・QA要件策定に貢献し、リリース判断まで横断的に担った。

freee人事労務へのモジュール統合 vs standalone展開を比較検討し、セキュリティ要件・開発速度・責務のシンプルさを総合評価してモジュール統合を選択。Design Docを作成し基盤チームとの合意形成を主導した。OEM提供元との仕様策定・外部コミュニケーションも担当し、スクラムのファシリテーション（スプリント計画・チケット策定・進捗管理）によって24卒・25卒を含むジュニアエンジニア2名のチームをリードしながら、自らもバックエンド・フロントエンドの実装を行った。

チーム全員が未経験のコードベース（人事労務）への統合と、前例のないパターンのOEM連携という二重の技術的不確実性、複雑なステークホルダー構造の中で、上長との1on1on1を自ら設計して構造的な支援を引き出し、ペアプロによるドメイン知識共有やリファインメントの整備で開発体制を再構築。上場企業であるOEM先との3者間で要件合意を形成し、リリースを実現した。

**AI活用（Claude Code）**

開発プロセス全体にAIを組み込み、3名チームでのアウトプット量を最大化した。

- **設計フェーズ**: Design Doc・PRDの壁打ちに活用。特に組織同期機能では、従業員データの同期パターン（新規/更新/削除/権限変更など複数の状態遷移）をAIと一緒に網羅的に洗い出した。AIなしでは見落としが多かった複雑なパターン分けを、設計段階で担保できた
- **実装フェーズ**: git worktreeと組み合わせてClaude Codeを複数エージェント並列運用。Storyのサブチケットに影響範囲を記載し、並列実行時に作業範囲が被らないかをチェックさせることでコンフリクトリスクを最小化
- **品質・セキュリティ**: Claude GitHub ActionsをCIに組み込みセキュリティ・コード品質を複数エージェントで自動チェック。CodeRabbitも併用。機密情報は社内LLM基盤のマスキングを前提に運用
- **チームへの展開**: `.claude`のskills/hooksを整備し、PRをsingle responsibilityで作成するcommandを共有。小粒度・目的明確なPRサイクルを回すことでジュニアのコードレビュー漏れを構造的に防いだ。テストはロジック・骨格を自分で書いてからAIで肉付けするスタイルを推奨し、AIへの過依存を防ぎながら根拠ある実装を維持。MCPサーバー（Storybook, Figma, GitHub, Slack, Confluence, Jira, 社内脆弱性診断ツール）とClaudeCode Marketplaceプラグインも積極採用し、DesignDocや仕様書を読み込んだ状態でコンテキスト収集を自動化

**技術的チャレンジ**

- OEMプロダクトのセキュリティ要件について、freee PSIRTチームとOEM提供元の間で落とし所を調整。3者間のセキュリティ基準を合わせながらリリースを実現
- OEM APIのRate Limit制約に対し、ワーカーを使った非同期処理を実装。大量の従業員データ連携を実現しつつ、ユーザー向けに進捗表示・ステータス管理でUXを維持

**スキル:** Ruby, Ruby on Rails, TypeScript, Terraform, Kubernetes, ArgoCD, AWS, GitHub Actions, Bugsnag, Datadog, Redash

## 自己PR

フルスタックエンジニアとして約5年の実務経験を持ち、バックエンド・インフラを中心に実装からアーキテクチャ設計・チームリードまで幅広く担当してきました。
