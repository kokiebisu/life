# `/fukushuu` 復習詰まり追記方式への切り替え — Design Spec

## TL;DR

`/fukushuu` を以下のように改修する:

- **AsIs:** セッションごとに独立した記録（実態として weakness.md ファイル / 復習用 Notion ページ）が分散
- **ToBe:** 各ノート（md + Notion）末尾の `## 🔁 復習で詰まったところ` セクションに**日付付き履歴**として追記
- **+α:** 自己評価（⭕/🔺/❌）を **Claude が面接官として判定**する方式に変更

これにより「どのノートをいつ詰まったか」が**ノート自身に記録される**ため、復習時の振り返りが一目でできる。weakness.md 方式は廃止。

---

## 背景

### 現状の課題

| 課題 | 詳細 |
|---|---|
| 復習履歴が分散 | `interview-prep/resilire/notes/YYYY-MM-DD/weakness.md` が日付ごとに散在し、章単位で「どこを何度詰まったか」が見えない |
| 自己評価のブレ | ユーザーが ⭕/🔺/❌ を選ぶ方式だと甘めの自己評価になりがち（「なんとなく言えた気がする」→ ⭕） |
| 詰まりの記録が粒度ゼロ | review-log.json には集約スコアしかなく、**個別の質問でどこを詰まったか**は失われる |
| 復習対象の追加コスト | weakness.md は除外リストにハードコードされており、メンテが必要 |

### ゴール

1. 各ノートに「どの質問でいつ詰まったか」が**累積記録**として残る
2. 自己評価が客観的になる（面接官目線で Claude が判定）
3. weakness.md という独立ファイルが不要になり、ノート構造がフラットになる
4. md と Notion 両方が常に同期される

## 設計

### アーキテクチャ概要

```
┌──────────────────────────────────────────────────────┐
│ /fukushuu セッション                                  │
│                                                      │
│ Step 4: 各質問を出題 → ユーザー回答 → Claude が判定   │
│   ├─ ✅ 正解   → 記録なし                              │
│   ├─ ⚠ 部分正解 → 「詰まった」リストに追加             │
│   └─ ❌ 詰まり  → 「詰まった」リストに追加             │
│                                                      │
│ Step 4.5: ノート末尾に追記                            │
│   ├─ md: ## 🔁 復習で詰まったところ に日付ブロック挿入│
│   └─ Notion: 同内容を notion-update-page で同期       │
│                                                      │
│ Step 5: review-log.json 更新（既存ロジック）          │
└──────────────────────────────────────────────────────┘
```

### 1. 追記対象とフォーマット

#### 対象ノート

`/fukushuu` の Step 3 で復習対象として選ばれた**全てのノート**。除外ルールは現行通り（`CLAUDE.md`、`README.md`、`roadmap.md`、`tracker.md`、`qa-bank.md`、`system-design-chapters.md`）。

> `weakness.md` の除外ルールは削除する（ファイル自体を廃止するため）。

#### MD フォーマット

セクション位置: **ファイル末尾**（既存セクションの後）。

```markdown
## 🔁 復習で詰まったところ

### 2026-04-30
- **Q: 3層スキーマの概念スキーマは誰の視点？**
  - 詰まった内容: 「DBMS視点」と答えてしまった
  - 正解ポイント: 開発者視点。論理設計（ER図・正規化）を行う層

### 2026-04-24
- **Q: POA と DOA の違いは？**
  - 詰まった内容: DOA を「処理先行」と逆に説明
  - 正解ポイント: DOA はデータ先行。データは業務本質を反映するため変化しにくい
```

#### フォーマットルール

| 項目 | ルール |
|---|---|
| 日付サブセクションの並び | **新しい順**（最新が一番上） |
| 同一日付内の Q の順 | 出題順（古い → 新しい） |
| Q の表記 | コーネル式キューの原文をそのまま `**Q: ...**` で記載 |
| 詰まった内容 | ユーザーの回答を要約（言い回しを直接引用しすぎない。要点のみ） |
| 正解ポイント | ノート本文から該当箇所を1〜2行で抽出 |
| 既存セクションへの追記 | `## 🔁 復習で詰まったところ` 直下に新しい日付ブロックを挿入（最新が上） |
| 全質問が ✅ のとき | **何も追記しない**（ノートをクリーンに保つ） |

### 2. 「詰まった」判定ロジック（面接官モード）

#### 質問単位の判定

| ユーザー回答 | 判定 | 追記 |
|---|:---:|:---:|
| 主要概念を正確に答えた | ✅ 正解 | なし |
| 主要概念は合っているが細部の値・定義・基準を欠落 | ⚠ 部分正解 | あり |
| 主要概念を取り違えた / 「わからない」「忘れた」と即答 | ❌ 詰まり | あり |

#### 全体評価の集約

| 質問の判定状況 | 全体評価 | review-log への記録 |
|---|:---:|---|
| 全質問が ✅ | ⭕ 完璧 | `confidence: "perfect"`、review_count +1 |
| ❌ なし、⚠ が1問以上 | 🔺 あいまい | `confidence: "fuzzy"`、review_count 据え置き |
| ❌ が1問以上 | ❌ 忘れた | `confidence: "forgot"`、review_count を 0 にリセット |

#### 面接官モードの基準

> **「面接で口頭説明した時に通用するか」を基準に判定する。**

- 用語の取り違え（POA と DOA を逆に説明 等）→ ❌
- 概念は合っているが具体値・定義の精度が落ちる → ⚠
- スラスラと正確に答えられた → ✅
- 「わからない」「忘れた」と即答 → ❌

#### ユーザーへの伝え方

- 全質問終了後、Claude が判定結果を提示する:
  > 面接官として判定: **❌ 忘れた**
  > 理由: POA/DOA を逆に説明したため

- AskUserQuestion での確認は **しない**（面接官モード = Claude が決める）
- ただしユーザーが異議を唱えたら（例: 「いや、今のは正解」）、再判定可。素直に評価を更新する

### 3. Notion 同期

#### 手順

```
1. md ノートを更新（## 🔁 復習で詰まったところ の挿入/追記）
2. ノートのフロントマターから notion_id を読む
3. notion_id あり:
   a. notion-fetch で既存本文を取得
   b. 末尾に同じ Markdown ブロックを追記
   c. notion-update-page (replace_content) で全文書き戻す
4. notion_id なし:
   a. notion-add.ts --db study で新規ページ作成
   b. 取得した notion_id をフロントマターに書き戻す
   c. 上記 3-a〜3-c と同じ手順で本文末尾に追記
```

#### 注意点

- Notion 側は md と完全一致させる（同じ Markdown ブロック）
- 既存本文を上書きせず、末尾に append する
- 失敗時はユーザーに通知する。md は更新済みでも Notion 失敗を明示し、再実行ガイドを示す

### 4. 既存 weakness.md の移行（同 PR で実施）

#### 対象ファイル

- `aspects/study/interview-prep/resilire/notes/2026-04-09/weakness.md`
- `aspects/study/interview-prep/resilire/notes/2026-04-10/weakness.md`
- `aspects/study/interview-prep/resilire/notes/2026-04-11/weakness.md`
- `aspects/study/interview-prep/resilire/notes/2026-04-13/weakness.md`

#### 移行手順

| ステップ | 内容 |
|:---:|---|
| 1 | 各 weakness.md を読み、内容を新フォーマットに変換 |
| 2 | 同ディレクトリの `note.md` 末尾に `## 🔁 復習で詰まったところ` を作成 |
| 3 | weakness.md の各項目を `### YYYY-MM-DD` サブセクションとして転記 |
| 4 | 自由形式で書かれた内容は `- **論点:** xxx` `- 詰まった内容: yyy` `- 正解ポイント: zzz` 形式に整形 |
| 5 | 対応する Notion ページ（`note.md` の `notion_id`）にも同内容を反映 |
| 6 | weakness.md を `git rm` で削除 |

#### 変換上の注意

- 各 weakness.md のヘッダー（例: `# Day 2: 復習で言えなかったこと`）は破棄
- フォーマットに当てはめにくい内容は箇条書きで転記し、可能な限り「Q / 詰まった内容 / 正解ポイント」構造に近づける
- `## ❓ 自分への質問（コーネル式キュー）` セクションが weakness.md にある場合は、対応する `note.md` に既にあるか確認し、なければ統合する

- 新規ノートは生成されなくなる

## エッジケース

| ケース | 対応 |
|---|---|
| ノートに `notion_id` がない | `notion-add.ts --db study` で新規作成し、フロントマターに書き戻す |
| Notion 更新失敗 | md は更新済み。ユーザーに「Notion 同期失敗、再実行してください」と通知 |
| 同日に二度復習しようとした | 現行通り対象外（`last_reviewed == today` フィルタ） |
| ユーザーが判定に異議 | 素直に再判定。質問単位 / 全体どちらも更新可 |
| 全質問 ✅ | 追記なし、`review_count +1`、`confidence: perfect` |
| キューが既存セクションにない | 現行通り Claude が自動生成して追記。詰まり追記もこの新キューに対して行う |
| ノートに `## ❓ 残った疑問・次回へ` がある（既存） | そのまま残す。`## 🔁 復習で詰まったところ` はその後ろに追加 |

## 検証方法

1. **既存 weakness.md 移行の検証**
   - 移行前後で4ファイル分の内容が `note.md` 末尾に正しく転記されているか目視確認
   - 対応する Notion ページにも同内容が反映されているか確認

2. **新フロー実行テスト**
   - サンプルとして章ノート1件で `/fukushuu` を実行
   - 故意に1問詰まり、1問部分正解、1問正解の回答をする
   - md と Notion 両方に正しいフォーマットで追記されるか確認
   - 全問正解パターンも実行し、何も追記されないことを確認

3. **`/study` スキル更新の検証**
   - 新規セッションで `## ❓ 残った疑問・次回へ` が生成されないことを確認
