# 学習ロードマップ

## フェーズ 1: 基礎固め

### 起業知識

- [ ] 法人設立の基礎（株式会社 vs 合同会社、設立手続き、費用）
- [ ] 事業計画書の書き方（ビジネスモデルキャンバス）
- [ ] 会計・経理の基礎（確定申告、帳簿の付け方）

### 法律

- [ ] 契約書の基本（業務委託契約、利用規約、プライバシーポリシー）
- [ ] 知的財産の基礎（著作権、特許、商標の違い）
- [ ] 個人開発者が知るべき法律リスク

### 技術

- [ ] tsumugi のアーキテクチャ深掘り（現状の設計を言語化）
- [ ] テスト戦略（ユニットテスト、E2Eテスト、テストピラミッド）
- [ ] リファクタリングの基本パターン

## フェーズ 2: 実践と深化

### 起業知識

- [ ] 資金計画（ブートストラップ vs 外部資金、ランウェイ計算）
- [ ] MVP の定義と検証方法（リーンスタートアップ）
- [ ] ユーザーインタビューの手法

### 法律

- [ ] 税務の基礎（法人税、消費税、インボイス制度）
- [ ] 労務の基礎（雇用契約、業務委託の違い、社会保険）
- [ ] スタートアップ向け資金調達の法的論点（株式、SAFE）

### 技術

- [ ] クラウドネイティブアーキテクチャ（コンテナ、オーケストレーション）
- [ ] CI/CD パイプラインの設計と最適化
- [ ] 可観測性（ログ、メトリクス、トレーシング）

## フェーズ 3: スケーリング

### 起業知識

- [ ] マーケティングの基礎（SEO、コンテンツマーケティング、SNS）
- [ ] 営業・セールスの手法（B2B、B2C の違い）
- [ ] プロダクトマーケットフィット（PMF）の測定方法

### 法律

- [ ] 資金調達時の法務（投資契約、株主間契約）
- [ ] 海外展開時の法律（越境EC、GDPR、各国規制）
- [ ] EXIT 戦略の法的側面（M&A、IPO）

### 技術

- [ ] システムのスケーラビリティ設計
- [ ] Staff+ エンジニアとしてのスキル（技術戦略、組織への影響力）
- [ ] 最新技術トレンドの評価と導入判断

## 進め方

- 各フェーズは順番に進める必要はない。興味や必要に応じて柔軟に取り組む
- 1つのトピックにつき「学ぶ → 要約する → 実践する」のサイクルを回す
- 学んだことは `aspects/diary/` や Issue に記録して振り返る
- 迷ったら `/ask:study` でメンターに相談する
