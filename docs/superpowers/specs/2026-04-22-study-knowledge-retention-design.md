# 学習ノート知識定着システム改善 — Design Spec

## 背景

`/study` スキルで学習ノートを作成し、`/fukushuu` スキルで忘却曲線ベースの復習を行うシステムが存在する。しかし以下の課題がある:

1. **コーネル式キューの欠如** — 既存ノート約30件に `❓ 自分への質問（コーネル式キュー）` が1件もない。`/fukushuu` のクイズがフォールバック（汎用的な問いかけ）に頼っており、復習の質が低い
2. **復習フィードバックが粗い** — y/n の二択で、「なんとなく覚えてた」を表現できない
3. **既存ノートの活用不足** — review-log.json に5件しかログがなく、大半のノートが復習対象として機能していない

## ゴール

- 既存30件のノートが即座に復習可能になる
- 今後のノートも自動でキュー付きで作成される
- 復習の粒度が細かくなり、「あいまい」なノートが適切な頻度で再出題される
- 面接対策（テックリード / シニアフルスタック）に直結する質問が含まれる

## 変更1: `/study` スキル改修 — セッション終了時のキュー自動生成

### 対象ファイル

`skills/study/SKILL.md` の Step 7（セッション終了）

### 変更内容

まとめ作成の後（手順2の後、手順3の前）にキュー生成ステップを追加する。

**追加フロー:**

1. Claude がノート内容（📝 ノート + 🔑 キーワード + 💡 まとめ）を分析
2. 3〜5問のコーネル式キューを生成
3. ユーザーに提示して確認（追加・削除・修正OK）
4. 確定したキューを MD の `❓ 自分への質問（コーネル式キュー）` セクションに書き込み
5. Notion にも同期（`replace_content` で全文反映）

### キュー生成方針

ノートの内容に応じて Claude が最適なバランスで質問タイプを選ぶ:

- **概念確認型:** 「〇〇とは？」「〇〇と〇〇の違いは？」
- **応用型:** 「このケースでどう設計する？」「なぜこの方法を選ぶ？」
- **判断力型:** 「〇〇のメリット・デメリットは？」「どういう条件で〇〇を使う？」

ユーザーの就活目標（テックリード / シニアフルスタック）を踏まえ、面接で問われそうな角度を優先する。

### MD フォーマット

`💡 まとめ` セクションの後、`❓ 残った疑問・次回へ` セクションの前に配置:

```markdown
## ❓ 自分への質問（コーネル式キュー）

1. B-Treeインデックスの計算量はO(?)で、なぜ「オール4の秀才型」と言えるか？
2. インデックスが適用されない5つのパターンを挙げよ
3. カーディナリティが高くてもインデックスが効かないケースとは？
```

## 変更2: `/fukushuu` スキル改修 — 3段階フィードバック

### 対象ファイル

`skills/fukushuu/SKILL.md`

### フィードバック3段階化

全問終了後の「覚えてた？ (y/n)」を以下に変更:

| 回答 | 意味 | review_count | 次の間隔 |
|------|------|-------------|---------|
| ⭕ 完璧 | 全問スラスラ答えられた | +1 | 通常テーブル通り |
| 🔺 あいまい | 方向性は合ってたが細部が怪しい | 変更なし | `interval_days` も変更なし。`last_reviewed` だけ今日に更新（同じ間隔でもう1回） |
| ❌ 忘れた | ほぼ思い出せなかった | 0にリセット | 1日後 |

復習間隔テーブルは変更なし:

| review_count | 間隔 |
|---|---|
| 0 | 即座 |
| 1 | 1日 |
| 2 | 3日 |
| 3 | 7日 |
| 4 | 14日 |
| 5 | 30日 |
| 6+ | 60日 |

### review-log.json の拡張

`confidence` フィールドを追加:

```json
{
  "aspects/study/software-engineering/notes/2026-04-22-1.md": {
    "last_reviewed": "2026-04-22",
    "interval_days": 3,
    "review_count": 2,
    "confidence": "perfect"
  }
}
```

`confidence` の値: `"perfect"` / `"fuzzy"` / `"forgot"`。既存エントリに `confidence` がない場合は後方互換として正常動作する。

### キュー不在ノートの処理改善

バックフィルで全ノートにキューが入る想定だが、万が一ない場合:

- 現行通り「何を学んだか説明してみて」でフォールバック
- 復習終了後に「このノートにキューがなかったので生成しますか？」と提案する

## 変更3: `/backfill-cues` スキル — 既存ノートへのキュー一括追加

### 対象ファイル

`skills/backfill-cues/SKILL.md`（新規作成）

### 動作フロー

1. `aspects/study/**/notes/*.md` を Glob で取得（CLAUDE.md / README.md / team/ 等は除外）
2. 各ノートを Read し、`❓ 自分への質問（コーネル式キュー）` セクションが既にあるかチェック
3. ないノートをリストアップしてユーザーに件数を報告
4. 1件ずつ処理:
   - ノートの内容（📝 ノート + 🔑 キーワード + 💡 まとめ）を読む
   - 3〜5問のキューを生成（`/study` と同じ生成方針）
   - MD の `💡 まとめ` の後、`❓ 残った疑問・次回へ` の前に挿入
   - 対応する Notion ページにも同期（フロントマターの `notion_id` を使い `notion-update-page` の `replace_content`）
5. 全件完了後、review-log.json をリセット（全ノートの `review_count` を 0、`interval_days` を 0 に戻す）

### review-log リセットの理由

キューが追加されたことで復習の質が変わる。全ノートを新鮮な状態で1回復習し直す方が、定着の起点として有効。

## 変更しないもの

- ノートの基本フォーマット（`❓ 自分への質問` セクションが追加されるだけ）
- 復習間隔テーブル
- `/fukushuu` のクイズ出題フロー（キューがある場合の動作は既存通り）
- Notion の DB スキーマ
