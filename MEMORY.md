## コーディングエージェント × PR の教訓

### ⚠️ コンフリクト防止（2026-04-01 学習）

**問題:** 実装プロンプトに「PR も作れ」を混ぜたら、エージェントが「main直プッシュ＋feature branch PR作成」を二重にやってコンフリクト発生。

**根本原因:** 実装とPR作成を一つのプロンプトで混ぜた。

**正しいやり方（分ける）:**
1. 実装: `claude --print 'タスク内容'`（PRは作らせない）
2. PR作成: 実装完了後に `claude --print '/pr'`（`/pr` コマンド単体で実行）

`/pr` はClaude Codeが管理するワークフロー。単体で使えばブランチ管理が適切に行われる。

## ! コマンドリレールール（2026-04-02 設定）

- メッセージの冒頭が `!` で始まる場合、Claude Code に `/command` としてリレーする
- 全チャンネル共通で適用
- 例: `!kondate` → `~/life` で `/kondate` を Claude Code 実行
- コマンド名は `!` を除いた部分をそのまま使う
